バージョニングポリシー

ModDL/MML の暫定的なバージョニングポリシーは以下の通りです：

  • バージョンは 3 つの整数値 x, y, z を . で連結した x.y.z で表現します。たとえば 1.23.4 など。
  • x の値をメジャーバージョンと呼びます。メジャーバージョンが変わるタイミングでは破壊的変更を行う可能性があります。
    • ただし、メジャーバージョンが 0 の間は全てのバージョンで破壊的変更の可能性があります。
  • y の値をマイナーバージョンと呼びます。破壊的変更を伴わない機能追加を行った場合は、原則的にこのバージョンを上げます。
  • z の値をパッチバージョンと呼びます。バグ修正のみを行った場合はこのバージョンを上げます。
  • メジャー/マイナーバージョンを上げる場合、それより下位のバージョンは 0 にリセットします。
  • バージョンを付与する対象は「ModDL/MML 処理系、標準ライブラリ、ドキュメントを併せたもの」です。たとえば処理系を更新せずライブラリだけを更新した場合も、処理系を含めた全体のバージョンを変更します。
